## [Manual Input] - Apr 17, 2026, 12:51 PM UTC

1. Use XML / Structural Tagging: Replace conversational and wordy transitions (e.g., 'The idea is to tell users...') with clean tags like <core_objective> and <ui_components_required>. This vastly reduces input token bloat while giving the LLM a clearer hierarchical understanding of your requirements.
2. Implement Role + Context Framing: Anchor the prompt by starting with an instruction like 'Act as a Senior UX/UI Engineer.' Establish a high-authority persona to immediately align the model's output tone, ensuring you get professional wireframes rather than generic advice.
3. Apply Skeleton / Scaffolded Prompting: To guarantee the output matches your frontend needs, provide a layout template in the prompt (e.g., `[Header] -> [Hero: Token Value Prop] -> [Features Grid] -> [Footer]`). This forces the AI to structure its response exactly how you intend to build the webpage.
4. Utilize Negative Prompting with Constraint Layering: Stack your requirements by priority and add strict negative constraints. For example, explicitly state: 'Do NOT generate HTML/CSS/JS code; ONLY provide the structural UX layout and text copy.' This prevents the model from wasting hundreds of output tokens on unusable code blocks.
